# Карта файлов и контрольных состояний vertical slice

## Границы и правила

Карта связывает события эталонного маршрута с файлами оригинальной PC-копии.
Пути относительны каталогу установки. Оригинальные `.KWN`, `.RWS`, executable и
`.sav` не входят в Git; в документе хранятся только размеры, hashes, object IDs
и наблюдаемое состояние.

Статус **подтверждён** означает прямой structural/import probe или изменение
save. Статус **вывод** означает наиболее вероятную связь по имени/структуре,
которую ещё нужно подтвердить object reference или runtime trace.

## Карта запуска и уровня

| Событие | Файлы | Роль | Статус |
|---|---|---|---|
| Запуск executable | `Asterix.exe`, `GameModule.elb`, `GAME.KWN` | Bootstrap, исполняемый gameplay module и global managers | Наличие/структура подтверждены |
| Заставки и главное меню | `LVL000/LVL00.KWN`, `LVL000/00LLOC00.KWN` … `04LLOC00.KWN`, `LVL000/WINAS/WINAS0.rws`, `WINAS1.rws` | Level 00 не содержит STR и является вероятным presentation/menu level | Вывод по структуре и порядку запуска |
| Global locale/UI | `00GLOC.KWN` … `04GLOC.KWN` | Пять global locale packs | Наличие подтверждено; язык каждого индекса не сопоставлен |
| Загрузка Gaul | `LVL001/LVL01.KWN` | 3 402 level objects: logic, groups, hooks, cameras, animations, cinematics, checkpoint | Подтверждено protected-level probe |
| Locale Gaul | `LVL001/00LLOC01.KWN` … `04LLOC01.KWN` | Пять locale packs по 6 350 байт | Подтверждено; индекс текущей русской локали не установлен |
| Статическая сцена | `LVL001/STR01_00.KWN` … `STR01_04.KWN` | Geometry, scene nodes, local textures, collision и sector-local objects | Подтверждено прямым импортом |
| Level audio | `LVL001/WINAS/WINAS0.rws` … `WINAS10.rws` | 11 non-speech banks | Файлы подтверждены; tracks исследуются в задаче 15 |
| Диалоги | `LVL001/WINAS/SPEECH/{0..4}/*_WIN*.RWS` | Пять параллельных наборов по 21 speech bank | Файлы подтверждены; language/line mapping не установлен |
| Engine cinematics | 4 `CKCinematicScene`, 14 `CKCinematicSceneData` внутри `LVL01.KWN` | Вступительные и обучающие sequences | Classes подтверждены; event-to-scene mapping остаётся неизвестным |
| Runtime checkpoint | `CKHkAsterixCheckpoint` 2/193, `CKGrpAsterixCheckpoint` 4/75 и life objects внутри `LVL01.KWN` | Восстановление маршрута после смерти | Classes/count и наблюдаемое восстановление подтверждены |
| Persistent save | `SAVE/AOXXL.sav` | Профиль/прогресс установки | Изменение на save boundary и стабильность после load подтверждены |

`LVL000` как menu-level — осознанный вывод, а не доказанная event reference.
Остальные файлы установки не включаются в срез только по совпадению общих имён.

## Sector transitions

Нумерация `CKSas` однобазовая: sector IDs 1–4 соответствуют проходимым
`STR01_00`–`STR01_03`. `STR01_04` не содержит geometry/collision и в переходах
не участвует.

| `CKSas` object | Связь | AABB первой стороны: low → high | AABB второй стороны: low → high |
|---:|---|---|---|
| 0 | sector 1 ↔ 2 | `(-18.050, -6.903, 50.479)` → `(-10.149, 10.870, 65.434)` | `(-6.097, -3.778, 51.134)` → `(1.140, 12.294, 61.889)` |
| 1 | sector 2 ↔ 3 | `(35.874, 0.866, 26.254)` → `(53.005, 16.104, 41.722)` | `(49.553, 0.866, 7.576)` → `(64.855, 14.264, 21.448)` |
| 2 | sector 3 ↔ 4 | `(125.098, 13.620, 107.263)` → `(130.307, 16.268, 113.968)` | `(133.775, 13.729, 108.045)` → `(140.340, 16.308, 114.101)` |

Эти AABB подтверждают последовательную streaming-цепочку, но сами по себе не
задают сюжетные названия участков или момент Stage 1 completion.

## Состояния до и после ключевых событий

| Событие | До | После | Persistent save | Свидетельство |
|---|---|---|---|---|
| Новая игра / передача управления | Menu/presentation state | Астерикс на стартовой тропе | Штатная запись ещё не наблюдалась | Take A5, прямое наблюдение |
| Обязательный разговор со шпионом | Управление игроком | Управление блокируется диалогом и возвращается | Hash не менялся в предварительном прогоне | Прямое наблюдение, повтор диалога после restore |
| Получение урона | Gameplay, health до удара | Hit reaction и уменьшенный HUD | Дисковая запись не наблюдалась | Take C |
| Смерть | Нулевое здоровье / потеря управления | Runtime restore до более ранней точки маршрута | `AOXXL.sav` не перезаписывался | Два прямых восстановления |
| Checkpoint restore | Позиция смерти | Маршрут перед шпионом; диалог может повториться | Используется runtime-state, не отдельный `.sav` | Прямое наблюдение и повтор маршрута Take B |
| Stage save boundary | Baseline hash `09b87e…` | After hash `a8316b…`, тот же размер | Файл перезаписан в 15:35:30 | Direct save metadata |
| Повторная загрузка | After snapshot на диске | Управление возвращено | Hash/mtime не изменились | Прямое наблюдение и повторный checksum |

Не создаются фиктивные «before fight» saves: PC-версия не записала их на диск,
а runtime checkpoint memory не извлекалась. Для regression эти состояния нужно
воспроизводить детерминированным input trace или новым версионируемым save новой
реализации, не переименованием одного `AOXXL.sav`.

## Правила воспроизводимости

1. Перед тестом копировать live save во внешний каталог и фиксировать SHA-256,
   размер и mtime.
2. Не подменять live save без отдельной резервной копии.
3. После события повторно вычислять hash даже при неизменном размере.
4. Runtime-only checkpoint отмечать отдельно от persistent save.
5. Не добавлять `.sav`, KWN/RWS, дампы памяти, видео или извлечённые игровые
   ресурсы в Git.
