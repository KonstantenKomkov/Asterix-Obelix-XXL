# Протокол записи эталонного поведения

## Назначение

Протокол задаёт воспроизводимую запись оригинальной PC-версии для задач 4–7. Видео и оригинальные сохранения хранятся только локально вне Git. В репозиторий добавляются журнал таймкодов, технические параметры и SHA-256 записей.

Запись используется только для сверки поведения, таймингов и визуального результата. Извлечение моделей, сцен, текстур, анимаций и звука выполняется напрямую из локальных исходных `.KWN`/`.RWS`; экранная запись не является источником игровых ресурсов и не блокирует задачи инвентаризации и импортёра.

## Требования к окружению

- Оригинальная PC-версия запускается через локальный Wine-prefix.
- Виртуальный рабочий стол Wine: 1024×768.
- Игра использует настройки по умолчанию, если журнал явно не указывает другое.
- На macOS отключены уведомления и автоматическое скрытие окна.
- Средство записи захватывает игровое окно целиком с курсором, если курсор виден в игре.
- Частота записи не ниже 60 FPS; кодек сохраняет читаемость HUD и покадровых событий.
- Запись не масштабируется и не обрезается после захвата.

## Локальная структура

Рекомендуемый каталог вне репозитория:

```text
asterix-reference/
  raw/
  saves/
  metadata/
```

Имена файлов:

```text
YYYY-MM-DD_pc_gaul_stage1_take-N.mov
YYYY-MM-DD_pc_menu_pause_death_take-N.mov
```

## Обязательные записи

### Take A — полный основной маршрут

Один непрерывный дубль:

1. запуск executable;
2. заставки и главное меню;
3. начало новой игры;
4. загрузка Gaul Stage 1;
5. получение управления Астериксом;
6. прохождение обязательного маршрута без секретов;
7. все обязательные бои и взаимодействия;
8. завершение Stage 1;
9. первая следующая штатная точка сохранения;
10. подтверждение сохранения и возврат к управлению или меню.

Запись нельзя склеивать. Неудачный дубль сохраняется только до получения успешного, после чего в журнале отмечается причина отклонения.

### Take B — состояния и восстановление

Отдельная непрерывная запись:

1. загрузка созданного сохранения;
2. пауза и продолжение;
3. получение урона;
4. смерть персонажа;
5. экран/анимация перезапуска;
6. восстановление с checkpoint;
7. выход в главное меню;
8. повторная загрузка сохранения.

### Take C — измерительные манёвры

На ровном безопасном участке записать отдельными повторениями:

- idle не менее 10 секунд;
- прямой бег не менее 10 секунд;
- полный разворот на месте или на минимальном радиусе;
- одиночный и двойной прыжок в профиль;
- падение с известного уступа;
- одиночные удары и доступную базовую серию без противника;
- получение одиночного удара от базового легионера;
- движение камеры по каждой доступной оси;
- приближение камеры к стене и выход из столкновения.

Каждый манёвр повторяется не менее трёх раз без изменения настроек.

## Журнал таймкодов

Для каждого принятого видео создать Markdown-файл по шаблону:

```markdown
# YYYY-MM-DD_pc_gaul_stage1_take-N

- SHA-256:
- Размер:
- Длительность:
- Разрешение:
- FPS:
- Кодек:
- Wine runtime:
- Настройки игры:

| Начало | Конец | Событие | Вход игрока | Наблюдение |
|---:|---:|---|---|---|
| 00:00.000 | 00:00.000 | | | |
```

Таймкоды указываются с точностью до кадра или миллисекунды. Переходы загрузки, потеря/возврат управления, начало/конец атак, прыжков, урона, смерти и checkpoint отмечаются отдельными строками.

## Контрольные сохранения

Оригинальные файлы сохранения не добавляются в Git. Для каждого локального сохранения в журнале фиксируются:

- понятное имя состояния;
- момент создания по таймкоду видео;
- относительное положение на маршруте;
- SHA-256 и размер;
- ожидаемое состояние после загрузки.

Минимальный набор:

- начало Stage 1;
- перед первым обязательным боем;
- после первого обязательного боя;
- перед завершением Stage 1;
- первая точка сохранения после Stage 1.

## Критерии приёмки задачи 4

Задача считается выполненной только когда:

- существуют принятые Take A, B и C;
- Take A покрывает весь выбранный vertical slice без склеек;
- в Take B видны смерть, checkpoint restore, pause/resume и повторная загрузка;
- Take C содержит по три повторения всех измерительных манёвров;
- для каждого видео зафиксированы SHA-256 и технические параметры;
- журнал содержит таймкоды всех обязательных событий;
- оригинальные видео и сохранения находятся вне Git;
- независимый review подтверждает читаемость HUD, отсутствие пропусков и пригодность материала для задачи 5.

## Текущий внешний блокер

Процессу Codex/Terminal требуется разрешение macOS **Screen & System Audio Recording** для захвата экрана и **Accessibility** для управления оригинальной игрой. Пока разрешения не выданы, протокол можно подготовить, но задача 4 не может быть закрыта.
