# Каркас импортёра

Каркас отделён от Flutter UI и состоит из переиспользуемой Dart-библиотеки и CLI:

```sh
fvm dart run bin/importer.dart inspect <input-file>
```

На этапе каркаса команда поддерживает только синтетический учебный контейнер `ASTX`. Он проверяет транспорт данных, little-endian чтение, контроль границ и формат ошибок, но не имитирует и не документирует неизвестные поля оригинального `.KWN`.

Ошибки выводятся в stderr как однострочный JSON с устойчивым полем `error`; при наличии контекста добавляются `path`, `offset` и `details`. CLI использует ненулевые exit codes для неверных аргументов, некорректного входа и ошибок файловой системы.

Обезличенная фикстура хранится как читаемый hex-текст в `test/fixtures/synthetic_container.hex`. Она создана специально для проекта и не содержит байтов оригинальной игры.
